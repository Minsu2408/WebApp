React 일반 변수 vs State 상세 정리
### 일반 변수 (let, const)
UI 미갱신: 값이 변해도 리액트가 인지하지 못해 화면이 리렌더링되지 않습니다.

데이터 휘발성: 함수형 컴포넌트가 리렌더링될 때 변수가 초기화되어 저장된 값이 사라질 수 있습니다.

### State (useState)
자동 리렌더링: 값이 변경되면 리액트가 변경을 감지하고 화면을 자동으로 새로고침하여 UI를 동기화합니다.

상태 유지: 컴포넌트가 다시 그려져도 현재의 상태 값을 기억하고 보존합니다.

성능 관리: 리액트는 여러 상태 변경을 일괄 처리하여 성능 최적화를 돕습니다.

### 결론
사용자의 입력이나 실시간 변경 사항처럼 화면에 즉시 반영되어야 하는 데이터는 반드시 State를 사용해야 합니다.
